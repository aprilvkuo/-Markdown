# df

使用df命令，这个命令比较常用，大家都很熟悉。问题是这种方法，有时候挂载点和挂载的卷不在同一行，使用脚本分析需要一点技巧的
# mount
使用mount命令，mount -l，这种方法的缺陷在于没有卷的大小，但是挂载点和挂载的卷在同一行

# 查看文件/etc/mtab。

原理是，每新挂载一个卷基本上都会更新这个文件的，那么自然可以通过这个文件来查看挂载点和挂职的卷。这种方法比mount -l稍微清晰了一点，但是，有时候是不可靠的。
